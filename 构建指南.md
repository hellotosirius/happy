## 方案对比

| 方案 | 优点 | 缺点 | 网络要求 | 推荐度 |
|------|------|------|----------|--------|
| GitHub Actions | 不受本地网络限制<br>配置简单<br>可重复使用 | 需要配置 GitHub | 仅需访问 GitHub | ⭐⭐⭐⭐⭐ |
| EAS Build | 官方方案<br>配置简单 | 需要稳定的网络<br>依赖 Google Cloud Storage | 需要访问 Expo 和 Google 服务 | ⭐⭐⭐ |

## 方案 A：GitHub Actions 云构建（推荐）

### 优势
- ✅ 构建在 GitHub 云环境中进行，不受本地网络限制
- ✅ 配置一次，永久使用
- ✅ 构建日志完整，便于排查问题
- ✅ 适合网络环境受限的情况

### 前置条件
- GitHub 账号
- Expo 账号（已有：smileok2006）

### 配置步骤

#### 第 1 步：创建 Expo Personal Access Token

1. 访问 Expo Token 管理页面：
   ```
   https://expo.dev/accounts/smileok2006/settings/access-tokens
   ```

2. 点击 **"Create Token"** 按钮

3. 填写 Token 信息：
   - Name: `github-actions-build`（或其他名称）
   - 点击创建

4. **立即复制生成的 token**（只会显示一次，请妥善保存）

#### 第 2 步：配置 GitHub Repository Secret

1. 访问仓库 Secrets 设置页面：
   ```
   https://github.com/hellotosirius/happy/settings/secrets/actions
   ```

2. 点击 **"New repository secret"** 按钮

3. 填写 Secret 信息：
   - Name: `EXPO_TOKEN`
   - Secret: 粘贴步骤 1 中复制的 token

4. 点击 **"Add secret"** 保存

#### 第 3 步：触发构建

1. 访问 GitHub Actions 页面：
   ```
   https://github.com/hellotosirius/happy/actions
   ```

2. 在左侧选择 **"Build Android APK"** workflow

3. 点击右侧的 **"Run workflow"** 按钮

4. 选择构建配置：
   - Branch: `main`
   - Build profile: `huawei`（默认）

5. 点击绿色的 **"Run workflow"** 按钮开始构建

#### 第 4 步：下载 APK

1. 等待构建完成（约 10-15 分钟）

2. 访问 Expo 构建页面：
   ```
   https://expo.dev/accounts/smileok2006/projects/happy/builds
   ```

3. 找到最新的构建记录

4. 点击下载 APK 文件

### 注意事项

- Token 只需要配置一次，后续可以重复使用
- 构建过程可以在 GitHub Actions 页面实时查看日志
- 如果构建失败，可以在日志中查看详细错误信息

## 方案 B：EAS Build 云构建

### 优势
- ✅ Expo 官方推荐方案
- ✅ 配置简单
- ✅ 自动管理签名证书

### 前置条件
- 稳定的网络连接（能够访问 Expo 和 Google Cloud Storage）
- Expo 账号已登录

### 构建步骤

#### 使用命令行构建

```bash
# 进入项目目录
cd D:\VTProject\MySpace\happy

# 执行构建命令
yarn build:android:huawei
```

#### 构建过程

1. 解析 CHANGELOG.md
2. 运行 TypeScript 类型检查
3. 连接到 Expo 服务器
4. 自动递增 versionCode
5. 使用远程 Android 证书
6. 压缩并上传项目文件
7. 在云端构建 APK

#### 下载 APK

构建完成后，访问：
```
https://expo.dev/accounts/smileok2006/projects/happy/builds
```

### 常见问题

**问题：上传失败，提示 TLS 连接错误**

```
Failed to upload to EAS Build
Reason: Client network socket disconnected before secure TLS connection was established
```

**解决方案：**
- 检查网络连接
- 尝试使用 VPN 或代理
- 或切换到方案 A（GitHub Actions）

## 方案 C：本地 Gradle 构建

### 优势
- ✅ 完全本地控制
- ✅ 不依赖云服务

### 前置条件

#### 1. 安装 Gradle

**手动下载方式：**

1. 下载 Gradle 8.14.3：
   - 腾讯云镜像：`https://mirrors.cloud.tencent.com/gradle/gradle-8.14.3-bin.zip`
   - 阿里云镜像：`https://mirrors.aliyun.com/macports/distfiles/gradle/gradle-8.14.3-bin.zip`

2. 解压到本地目录（如：`D:\Tools\gradle-8.14.3-bin\gradle-8.14.3`）

#### 2. 配置 JDK

确保使用 JDK 17 或更高版本：

```bash
# 检查 Java 版本
java -version

# 应该显示类似：
# openjdk version "17.0.17" 2025-10-21
```

如果版本不正确，设置 JAVA_HOME 环境变量：
```bash
set JAVA_HOME=D:\JDK\Eclipse Adoptium\jdk-17.0.17.10-hotspot
```

#### 3. 配置 Android SDK

确保已安装 Android SDK，路径如：`D:\Android\Sdk`

#### 4. 生成原生 Android 项目

```bash
# 进入项目目录
cd D:\VTProject\MySpace\happy

# 生成原生项目
npx expo prebuild --platform android --clean
```

#### 5. 配置本地属性

创建或编辑 `android/local.properties` 文件：

```properties
sdk.dir=D\:\\Android\\Sdk
ndk.dir=D\:\\Android\\Sdk\\ndk\\27.0.12077973
```

#### 6. 配置 NDK 版本

编辑 `android/build.gradle` 文件，在 `apply plugin` 之前添加：

```gradle
ext {
    ndkVersion = "27.0.12077973"
}
```

#### 7. 配置镜像源（可选，用于加速下载）

编辑 `android/build.gradle` 文件，在 repositories 块中添加阿里云镜像：

```gradle
buildscript {
  repositories {
    maven { url 'https://maven.aliyun.com/repository/google' }
    maven { url 'https://maven.aliyun.com/repository/public' }
    google()
    mavenCentral()
  }
}

allprojects {
  repositories {
    maven { url 'https://maven.aliyun.com/repository/google' }
    maven { url 'https://maven.aliyun.com/repository/public' }
    google()
    mavenCentral()
    maven { url 'https://www.jitpack.io' }
  }
}
```

### 构建步骤

#### 构建调试版本（Debug）

```bash
# 进入 android 目录
cd D:\VTProject\MySpace\happy\android

# 设置 JAVA_HOME（如果需要）
set JAVA_HOME=D:\JDK\Eclipse Adoptium\jdk-17.0.17.10-hotspot

# 使用本地 Gradle 构建
D:\Tools\gradle-8.14.3-bin\gradle-8.14.3\bin\gradle.bat assembleDebug
```

#### 构建发布版本（Release）

```bash
# 使用本地 Gradle 构建
D:\Tools\gradle-8.14.3-bin\gradle-8.14.3\bin\gradle.bat assembleRelease
```

### 输出位置

构建成功后，APK 文件位于：

- **Debug 版本**：`android/app/build/outputs/apk/debug/app-debug.apk`
- **Release 版本**：`android/app/build/outputs/apk/release/app-release.apk`

### 常见问题

**问题 1：Gradle 下载失败**

```
Could not get resource 'https://services.gradle.org/distributions/gradle-8.14.3-bin.zip'
```

**解决方案：**
手动下载 Gradle 并配置本地路径（见前置条件）

**问题 2：NDK 版本不匹配**

```
NDK from ndk.dir had version [X] which disagrees with android.ndkVersion [Y]
```

**解决方案：**
修改 `android/build.gradle` 中的 `ndkVersion` 为已安装的版本

**问题 3：依赖下载失败（TLS 错误）**

```
The server may not support the client's requested TLS protocol versions
Remote host terminated the handshake
```

**解决方案：**
1. 配置阿里云镜像源（见配置步骤）
2. 配置 SSL/TLS 参数（编辑 `android/gradle.properties`）：
   ```properties
   org.gradle.jvmargs=-Xmx2048m -XX:MaxMetaspaceSize=512m -Dhttps.protocols=TLSv1,TLSv1.1,TLSv1.2,TLSv1.3
   ```
3. 如果仍然失败，切换到方案 A（GitHub Actions）

## 常见问题

### Q1: 为什么推荐使用 GitHub Actions？

**A:** GitHub Actions 在云端构建，不受本地网络环境限制。如果您的网络环境无法访问 Google 服务或存在 TLS 连接问题，GitHub Actions 是最可靠的选择。

### Q2: 构建需要多长时间？

**A:**
- GitHub Actions：10-15 分钟
- EAS Build：10-15 分钟
- 本地 Gradle：首次构建 15-30 分钟，后续 5-10 分钟

### Q3: 如何查看构建日志？

**A:**
- **GitHub Actions**：在 Actions 页面点击构建任务查看实时日志
- **EAS Build**：在 Expo 网站查看构建日志
- **本地 Gradle**：直接在终端查看输出

### Q4: APK 文件在哪里？

**A:**
- **GitHub Actions / EAS Build**：在 Expo 网站下载
- **本地 Gradle**：在 `android/app/build/outputs/apk/` 目录

### Q5: 如何解决网络问题？

**A:**
1. 优先使用 GitHub Actions（不受本地网络限制）
2. 配置镜像源（阿里云、腾讯云）
3. 使用 VPN 或代理
4. 更换网络环境（如使用手机热点）
